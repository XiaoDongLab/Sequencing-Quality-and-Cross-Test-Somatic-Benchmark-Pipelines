# Sequencing Quality Assessment and Cross-Test Somatic Benchmark Pipelines

This repository contains two independent analytical workflows:

1. `Sequencing Metrics Pipeline`: quantifies coverage breadth, coverage uniformity, read-depth structure, chromosome-scale coverage variation, and allele-fraction distributions from alignment data.
2. `Cross-Test Somatic Benchmark Pipeline`: evaluates a somatic call set defined relative to a bulk sample from the same tissue against a second unique-mutation call set defined relative to another tissue or cell-line comparator, using matched germline inputs and a pseudo-false-positive reference.

# Pipeline 1: Sequencing Metrics

## Biological and Analytical Scope

This workflow summarizes sequencing quality from a BAM file and estimates allele fractions at user-specified genomic sites. By default, the allele-fraction component uses `hSNP.bed` in each sample directory. That default is a convenience choice only; users should replace it with a study-specific site file whenever a different locus set is scientifically appropriate.

## Quantities Reported

- `Reads-versus-coverage`: a downsampling analysis that measures how many genomic bases are covered as the number of retained reads increases. In the current implementation, each BAM is downsampled from 20 million to 400 million reads in 20-million-read increments.
- `Lorenz curve and Gini coefficient of coverage`: summaries of coverage uniformity across the genome. The Lorenz curve compares the cumulative fraction of sequenced bases with the cumulative fraction of the genome covered, and the Gini coefficient provides a scalar summary of that inequality.
- `Read-depth distribution`: the per-base read-depth distribution in the observed BAM. This differs from the reads-versus-coverage analysis: read-depth distribution describes the final depth profile, whereas reads-versus-coverage describes how coverage breadth changes under downsampling.
- `Chromosome-window coverage`: coverage summarized in consecutive `1 Mb` windows across `chr1` to `chr22` and `chrX`.
- `Observed allele-fraction distribution`: allele fractions computed from `FORMAT/AD` values generated by `bcftools mpileup` at user-supplied sites. The site file may be a `VCF`, `VCF.GZ`, `BED`, or `BED.GZ` file.

## Required Inputs Per Sample

- `<sample_id>_WGNS_filtered.bam`: alignment file used for downsampling, depth calculations, coverage summaries, and mpileup-based allele-fraction estimation
- A site file for allele-fraction analysis:
  - Default: `hSNP.bed`
  - Supported formats: `VCF`, `VCF.GZ`, `BED`, `BED.GZ`
  - BED files must contain at least five columns in this order: `chrom`, `start`, `end`, `ref`, `alt`

Additional required inputs:

- A text file with one sample ID per line
- Per-sample directory structure: `../lab_data/<sample_id>/`
- Reference files:
  - `reference.fa`
  - `reference.fa.fai`

## Running the Workflow

Minimal command:

```bash
bash run_sequencing_metrics_pipeline.sh sample_ids.txt
```

Explicit command with user-defined values:

```bash
bash run_sequencing_metrics_pipeline.sh \
  sample_ids.txt \
  8 \
  ../lab_data \
  ./sequencing_metrics_results \
  /path/to/reference.fa \
  hSNP.bed
```

Argument definitions:

- `sample_ids.txt`: sample list file
- `8`: number of threads
- `../lab_data`: parent directory that contains per-sample folders
- `./sequencing_metrics_results`: output directory for merged tables and plots
- `/path/to/reference.fa`: reference FASTA used for mpileup and chromosome lengths
- `hSNP.bed`: allele-fraction site file relative to each sample directory; an absolute path or a VCF file can also be supplied

Important note on defaults:

- The default reference path in the script is a convenience placeholder and should not be used without verification.
- The default allele-fraction site file `hSNP.bed` is not universally appropriate; users should provide the site file that matches the experimental design and analytical question.

## Outputs

Per-sample intermediate files include downsampling summaries, Lorenz data, read-depth distributions, chromosome-window coverage tables, and mpileup-derived allele-fraction tables.

Merged summary tables are written to `sequencing_metrics_results/bridge_tsv/`.

Plots are written to `sequencing_metrics_results/plots/`, including:

# Pipeline 2: Cross-Test Somatic Benchmark

## Biological and Analytical Scope

This workflow is designed for settings in which the primary somatic mutation call set is defined as mutations that are unique relative to a bulk sample from the same tissue. The benchmark then evaluates that primary unique-mutation set against a second unique-mutation set generated using a different bulk tissue or cell-line comparator. Germline call sets from the sample and from the comparison tissue or cell line are used to establish the expected variant background, and a broad germline-mark reference is used to define a pseudo-false-positive background. The aim is to obtain interpretable pseudo-true-positive rates, pseudo-false-positive rates, mixture summaries, and burden estimates under this cross-comparison design.

## Required Inputs

All VCF inputs must be compressed and indexed.

- `--sample SAMPLE`: sample identifier used in output filenames
- `--callable 2500000000`: number of callable bases used as the denominator for rate calculations
- `--somatic somatic.vcf.gz`: primary somatic call set under evaluation. In a standard somatic-calling workflow for this analysis, this file should contain mutations that are unique relative to a bulk sample from the same tissue as the primary sample.
- `--cross cross_test_somatic.vcf.gz`: comparison somatic call set. In this workflow, this file should contain mutations that are unique relative to a bulk sample from another tissue or from a comparison cell line. Use of a well-characterized cell line as the comparison tissue or comparator source is strongly recommended.
- `--germS germline_sample.vcf.gz`: germline call set associated with the primary sample
- `--germT germline_test.vcf.gz`: germline call set associated with the comparison tissue or comparison cell line
- `--fpref pseudo_fp_reference.vcf.gz`: reference VCF that includes as comprehensively as possible the germline mutation marks that may be present in the sample. This file is used to define a pseudo-false-positive background set.
- `--out results`: output directory

Optional inputs:

- `--filter PASS`: retain only records with the specified FILTER value
- `--ref reference.fa`: reference FASTA used by `bcftools norm`
- `--reflen 3137300923`: total reference length used in burden scaling

Important note on defaults and example values:

- The example value `2500000000` for `--callable` is not a recommended universal default. Users should calculate callable bases for their own data.
- The default `--reflen 3137300923` is a convenience value and should be replaced when the relevant reference length differs.

Practical guidance on file construction:

- The scientific meaning of this benchmark depends on how the input VCFs were generated.
- The `--somatic` and `--cross` inputs should be produced with comparable calling and filtering logic so that differences reflect biology and comparator choice rather than pipeline asymmetry.
- The comparison input used for `--cross` should preferably be based on a well-characterized cell line or other well-controlled comparator, because poorly characterized comparison material can weaken the interpretability of pseudo-true-positive and pseudo-false-positive estimates.
- The `--fpref` input should be intentionally inclusive with respect to possible germline marks; an incomplete file will reduce its usefulness as a pseudo-false-positive background reference.

## Running the Workflow

```bash
bash run_sm.sh \
  --sample SAMPLE \
  --callable 2500000000 \
  --somatic somatic.vcf.gz \
  --cross cross_test_somatic.vcf.gz \
  --germS germline_sample.vcf.gz \
  --germT germline_test.vcf.gz \
  --fpref pseudo_fp_reference.vcf.gz \
  --out results
```

## Outputs

Tables:

- `<sample>.x.tsv`: primary benchmark summary table for the unique-mutation comparison
- `<sample>.m.tsv`: germline-derived mixture summary table
- `<sample>.snv_xy.tsv`: SNV pseudo-false-positive versus pseudo-true-positive coordinates under the cross-comparison design
- `<sample>.ind_xy.tsv`: indel pseudo-false-positive versus pseudo-true-positive coordinates under the cross-comparison design
- `<sample>.mix.tsv`: mixture summary metrics
- `<sample>.burden.tsv`: corrected somatic burden estimates

Plots:

- `<sample>.snv_tp_vs_fp.pdf`
- `<sample>.ind_tp_vs_fp.pdf`
- `<sample>.mix_inferred.pdf`
- `<sample>.burden_snv.pdf`
- `<sample>.burden_ind.pdf`

Temporary `bcftools isec` files are written to `results/tmp/`.
